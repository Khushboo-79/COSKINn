# COSKINn — Tax / Finance Panel Build Plan (React + Tailwind)

**Repo path:** `apps/internal-panel/src/modules/finance`
**Stack:** React + Tailwind CSS · shared `packages/ui` component library · React Query · React Hook Form + Zod · shared route shell (login, nav, layout) inside `apps/internal-panel`
**Depends on backend:** `auth/RBAC` + 2FA (`02-BACKEND-BUILD-PLAN.md` Day 9), `tax/HSN` module (Day 53), `GST invoice/credit-debit notes` module (Day 54), `finance reports` module (Day 55), `settlement/reconciliation` module (Day 56)
**Connects to panels** (per `00-PANEL-CONNECTIVITY-MAP.md` §2):
- **Sends to →** Admin Panel (P&L, GST dashboards), Auditor Panel (tax/payment reports)
- **Receives from ←** Order Management Panel (order + invoice data), Payments module (Razorpay settlements), Returns module (refund data), Wallet/Rewards modules (liability ledgers)

All of the above happens only through backend APIs — this panel never calls Order Management or Order data sources directly.

Per the build-order dependency chain (`00-PANEL-CONNECTIVITY-MAP.md` §4), this panel is Phase 5 work — it can't be built for real until Order Management (invoices), Payments (Razorpay settlement events), Returns/Refunds, and the Loyalty ecosystem (wallet/reward liability) are all live, since every screen here aggregates data those modules produce.

**Compliance/data-integrity note:** this module is one of two internal panels (with Admin) requiring mandatory 2FA per the non-functional targets in `02-BACKEND-BUILD-PLAN.md` — the login flow for this panel must not allow bypass even for RBAC-elevated roles. Every report screen is read/aggregate-only against source-of-truth tables owned by other modules (orders, payments, returns, wallet); this panel must never allow direct mutation of those underlying records — credit/debit notes and reconciliation flags are the only new financial records this panel is authorized to create.

**Definition of done:** every report in PRD/Build Form §6.6 (GST/HSN management, tax invoices, credit/debit notes, sales & GST reports, Razorpay settlement reconciliation, COD reconciliation, wallet & reward-points liability ledger, P&L report) is live against real backend data with no mocked figures, 2FA is enforced on every login to this module, and every export action is captured in the audit log visible to the Auditor Panel.
